Data Dictionary

IntelliScan CRM & OCR Engine Architecture


This document defines the highly detailed schema structure, constraints, and data definitions for the core operational tables of the IntelliScan project database. It explicitly outlines the constraints required to maintain referential integrity across the multi-tenant architecture.

1. Table: Workspaces

Description: Core architectural tenant representing an enterprise or company isolating system data.

Field Name Data Type Size Constraint Description
workspace_id UUID 36 PK Unique identifier for the tenant company.
company_name VARCHAR 255 NN Official business name mapped to the tenant.
max_users INT 4 - Current seat limit for users (Default: 5).
quota_scans INT 4 - Monthly limits for Gemini Vision AI parsing calls.
created_at DATETIME 8 NN Timestamp for the creation of the Workspace context.

2. Table: Users

Description: Account and authentication matrix for accessing the dashboard.

Field Name Data Type Size Constraint Description
user_id UUID 36 PK Primary authentication identity.
workspace_id UUID 36 FK Binds the user directly to the Workspace enterprise level.
email_address VARCHAR 255 NN, UQ Login credentials ensuring globally unique values.
password_hash VARCHAR 255 NN Secure Bcrypt hashing generated array for standard storage.
rbac_role VARCHAR 50 NN Authorization string defining scope ('Owner', 'Admin', 'User').
is_active BOOLEAN 1 - Flag determining access standing (Default: True).

3. Table: Contacts (Leads)

Description: Primary entity mapped from extracted Business Card OCR processes encapsulating lead records.

Field Name Data Type Size Constraint Description
contact_id UUID 36 PK Definitive primary key for a successfully uploaded lead.
workspace_id UUID 36 FK Tenant ownership preventing scope leaks.
first_name VARCHAR 100 - Extracted First Name from structured Gemini model output.
last_name VARCHAR 100 - Extracted Last Name resolving empty arrays.
email_address VARCHAR 255 - Primary mechanism for matching automated email CRON sequences.
company_title VARCHAR 255 - Entity representation or organizational role of the lead.
pipeline_stage VARCHAR 50 NN CRM Tracking (Default: 'New Lead').
confidence_score DECIMAL (5,2) NN The certainty margin outputted by the LLM logic check.

4. Table: Scanned_Images

Description: Raw storage matrix that handles the heavy byte transactions before OCR application.

Field Name Data Type Size Constraint Description
image_id UUID 36 PK Identifier mapped to the file uploaded from the mobile device.
contact_id UUID 36 FK Associates the raw blob logically to the refined Contact mapping.
s3_bucket_url VARCHAR 500 NN Absolute cloud linkage for image delivery back to the frontend.
raw_ocr_json TEXT - - Massive string capturing the absolute unmodified return from Gemini.

5. Table: Events

Description: Grouping logic separating different trade-shows and networking functions.

Field Name Data Type Size Constraint Description
event_id UUID 36 PK Top-level scope variable.
workspace_id UUID 36 FK Ties the event strictly to the parent Workspace account.
event_name VARCHAR 255 NN The UI displayed string describing the convention or meeting.
start_date DATE 8 NN Enforces when scanning limits/QR codes activate.
custom_qr_url VARCHAR 500 - The generated public-facing lead ingestion portal for this event.

6. Table: Email_Campaign_Nodes

Description: Background task processing queue parsed hourly to dispatch SMTP relays based on date checks.

Field Name Data Type Size Constraint Description
campaign_id UUID 36 PK Core tracking element for a single outgoing step.
target_contact_id UUID 36 FK The ultimate recipient of the email transmission.
trigger_delay_days INT 4 - How long the daemon should wait before attempting execution (e.g. Day 2, Day 7).
html_rendered_body TEXT - NN The generated and customized HTML text block prepared for NodeMailer handling.
delivery_status VARCHAR 50 NN Execution map tracking state ('Queued', 'Sent', 'Bounced').

7. Table: Calendar_Bookings

Description: Public widget interactions validating available logical time slots without overlaps.

Field Name Data Type Size Constraint Description
booking_id UUID 36 PK Master key establishing the confirmed reservation.
host_user_id UUID 36 FK Ties the booking block to the User holding the master schedule.
guest_email VARCHAR 255 NN Extrapolated text required to fire ICS attachment emails.
start_block DATETIME 8 NN Lock defining the earliest timestamp of the booking period.
end_block DATETIME 8 NN Lock defining the termination of the booking period (preventing overlaps).
meeting_url VARCHAR 500 - Bridge connection URL linking to Google Meet / Zoom auto-generation.
