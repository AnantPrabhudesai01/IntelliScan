# IntelliScan: Comprehensive Project Analysis & Documentation

## 1. Fully Detailed Project Documentation

### Executive Overview
IntelliScan is a full-stack, enterprise-grade application for scanning, extracting, and managing professional contacts (digital rolodex) with AI integration. The project bridges a React-based frontend (Vite) and a Node.js/Express backend, heavily relying on Google's Gemini AI for OCR extraction. It supports multi-tenant workspaces, real-time collaboration, and varying roles (User, Business Admin, Super Admin).

### Technology Stack
**Frontend:**
- **Framework:** React 19 (via Vite)
- **Routing:** React Router v7
- **Styling:** Tailwind CSS v3.4 + PostCSS
- **Icons:** Lucide React (strictly enforced)
- **State Management:** React Context API
- **Data Fetching:** Axios / native Fetch API
- **Real-time:** Socket.io-client

**Backend:**
- **Runtime:** Node.js
- **Framework:** Express.js v5.2
- **Database:** SQLite3 (Local `database.sqlite` file)
- **Authentication:** JWT (JSON Web Tokens) with bcryptjs
- **AI Integration:** Google Generative AI (`@google/generative-ai`)
- **Real-time:** Socket.io Server

### Architecture Diagram

The system follows a classic client-server model extended with WebSocket capabilities.

```mermaid
graph TD
    Client[React Frontend / Vite] -->|REST API & Auth Token| API[Node.js / Express Backend]
    Client <-->|WebSockets / Socket.io| RT[Real-time Server]
    API -->|Base64 Image + Prompt| Gemini[Google Gemini AI API]
    API <-->|SQL Queries| DB[(SQLite Database)]
```

### Database Schema Explanation (SQLite)
- **users:** Stores credentials, roles (`user`, `business_admin`, `super_admin`), and workspace assignments.
- **sessions:** Tracks active logins, IP addresses, location, and device info.
- **contacts:** The core entity. Stores extracted business card data, confidence scores from AI, scanner ID, and metadata.
- **events & ai_drafts:** Used for the CRM side, tracking networking events and AI-generated follow-up draft emails.
- **workspace_chats:** Stores real-time chat messages broadcast via Socket.IO.
- **engine_config & model_versions:** Stores OCR engine preferences and thresholds for super admins to tune performance.

### Environment Variables
**Backend (`.env`):**
- `PORT=5000`
- `JWT_SECRET=supersecret_intelliscan_key_123`
- `GEMINI_API_KEY=<Google AI Key>`

### Deployment & Local Setup
1. **Backend Integration:** 
   - `cd intelliscan-server` -> `npm install`
   - Ensure a valid `GEMINI_API_KEY` is in `.env`.
   - Start with `npm run start` or `npm run dev` (Runs on `localhost:5000`).
2. **Frontend Run:** 
   - `cd intelliscan-app` -> `npm install`
   - Run `npm run dev` (Runs Vite server).
   - Vite proxies `/api` to `http://127.0.0.1:5000`.

## 4. Detailed Gap Analysis – What is Lacking

**Overall Health:** The architecture is modern and feature-rich, but suffers from inconsistencies typical of rapid prototyping and transitioning from static mocks to a live backend.

### 🔴 Critical Issues (The "Nothing is Connected" Problem)
1. **Inconsistent API Base URLs:** 
   - **The Bug:** `vite.config.js` correctly proxies `/api` to `http://127.0.0.1:5000`. However, throughout the frontend (e.g., `SharedRolodexPage.jsx`, `SettingsPage.jsx`), developers have hardcoded `fetch('http://localhost:5000/...')`.
   - **Impact:** Hardcoded URLs bypass the Vite proxy. If CORS is slightly off, or if the user tests from their phone on the same WiFi (`192.168.x.x`), these hardcoded calls will fail entirely, yielding "Network Error" or appearing completely disconnected.
   - **Fix:** Replace all instances of `http://localhost:5000/api/...` with relative paths `/api/...` or use a globally configured Axios instance.

2. **Google Gemini API Dependency Not Configured:** 
   - **The Bug:** The core `/api/scan` endpoint relies completely on `process.env.GEMINI_API_KEY`. If the user does not supply this (which usually isn't committed to git), the endpoint throws a 500 error. Because the frontend relies on this scanning, failure causes the core app function to break.
   - **Fix:** Add robust frontend fallback/mock behavior if the API returns 500, OR bundle a mock AI endpoint for testing. Provide clear UI alerts if keys are missing.

3. **Critical Privilege Escalation (Upgrade Bug):**
   - **The Bug:** If a user clicks the "Upgrade" button, the system incorrectly routes them to the Super Admin dashboard (or elevates their view privileges), allowing them to see everything a Super Admin sees. 
   - **Fix:** Remove any frontend state manipulation that alters the user's `role`. Ensure backend JWTs are the sole source of truth for routing guards. Upgrading should strictly adjust workspace tiers, not user roles.

### 🟠 High Severity Issues
4. **Authentication State Desync:** 
   - **The Bug:** Auth is handled synchronously by checking `localStorage.getItem('user')` on page load in isolated files, rather than relying strictly on a rigorous `AuthProvider`. If a token expires, the Express backend throws 401 or 403, but the frontend lacks global interceptors to gracefully boot the user back to `/sign-in`.
   - **Fix:** Implement an Axios interceptor to catch `401 Unauthorized` responses globally and clear local storage / redirect to sign in.

5. **Disconnected Enterprise Feedback Loop:**
   - **The Bug:** When an Enterprise user submits platform feedback, there is no connection to the Super Admin dashboard. The Super Admin cannot see the feedback or take action on it.
   - **Fix:** Implement a dedicated API pipeline and SQLite table (`platform_feedback`) fetching directly into the `/admin/dashboard` view, complete with action buttons (Resolve/Reply).

6. **Socket.IO Hardcoding:**
   - **The Bug:** `SharedRolodexPage.jsx` initializes with `io('http://localhost:5000')`.
   - **Fix:** Change to `io()` so it infers the host, or feed from an environment variable.

### 🟡 Medium & Low Severity Issues
5. **Missing Input Sanitization & Validation:** Express uses `req.body` blindly in registration and contact creation. Add `zod` or `joi` to validate backend payload shapes to prevent injection or malformed data breaks.
6. **Partial Mock Data Reliance:** The *Technical Addendum* insists heavily on a `mockContacts` structure. Some pages may still be mapping this static array instead of the real `ContactContext` data from SQLite.
7. **Missing Error Boundaries:** React will crash completely if the UI encounters an unhandled exception in rendering null data (e.g., a contact with an unexpectedly missing ID mapping).

### 🔵 Missing Core Features
8. **Personalized Automated Chatbot:** Both Personal and Enterprise users currently lack a dedicated support/assistant chatbot.
   - **Fix:** Implement a persistent floating chat interface leveraging the Gemini API. The system prompt must be dynamically injected with the user's context (tier, recent scans, problems) so the AI provides highly customized solutions rather than generic answers.

// Auth Middleware
const authenticateToken = (req, res, next) => {
  const authHeader = req.headers['authorization'];
  const token = authHeader && authHeader.split(' ')[1];
  if (token == null) return res.sendStatus(401);

  jwt.verify(token, process.env.JWT_SECRET, (err, user) => {
    if (err) return res.sendStatus(403);
    req.user = user;
    next();
  });
};

app.post('/api/auth/register', async (req, res) => {
  const { name, email, password, role } = req.body;
  try {
    const hashedPassword = await bcrypt.hash(password, 10);
    const userRole = role || 'user';
    
    db.run(
      'INSERT INTO users (name, email, password, role) VALUES (?, ?, ?, ?)',
      [name, email, hashedPassword, userRole],
      function (err) {
        if (err) {
          if (err.message.includes('UNIQUE')) {
            return res.status(400).json({ error: 'Email already exists' });
          }
          return res.status(500).json({ error: err.message });
        }
        
        const token = jwt.sign(
          { id: this.lastID, email, name, role: userRole }, 
          process.env.JWT_SECRET, 
          { expiresIn: '24h' }
        );
        
        res.status(201).json({ 
          message: 'User registered successfully',
          token,
          user: { id: this.lastID, name, email, role: userRole }
        });

    res.status(500).json({ error: 'Server error' });

// Login
app.post('/api/auth/login', (req, res) => {
  const { email, password } = req.body;
  
  db.get('SELECT * FROM users WHERE email = ?', [email], async (err, user) => {
    if (err) return res.status(500).json({ error: err.message });
    if (!user) return res.status(401).json({ error: 'Invalid email or password' });

    const validPassword = await bcrypt.compare(password, user.password);
    if (!validPassword) return res.status(401).json({ error: 'Invalid email or password' });

    const token = jwt.sign(
      { id: user.id, email: user.email, name: user.name, role: user.role }, 
      process.env.JWT_SECRET, 
      { expiresIn: '24h' }
    );

    // Session tracking on login
    const deviceInfo = req.headers['user-agent'] || 'Unknown Device';
    const ipAddress = req.ip || 'Unknown IP'; 
    const location = 'Unknown Location'; // Placeholder, could use a geo-ip service

    db.run(
      'INSERT INTO sessions (user_id, token, device_info, ip_address, location, is_active) VALUES (?, ?, ?, ?, ?, 1)',
      [user.id, token, deviceInfo, ipAddress, location],
      function(sessionErr) {
        if (sessionErr) {
          console.error('Error inserting session:', sessionErr.message);
          // Log the error but don't prevent login if session tracking fails
        }

          message: 'Logged in successfully',
          token,
          user: { id: user.id, name: user.name, email: user.email, role: user.role }

    );
  });
});

app.post('/api/scan', authenticateToken, async (req, res) => {
  try {
    const { imageBase64, mimeType } = req.body;
    
    if (!imageBase64) {
      return res.status(400).json({ error: 'No image provided' });
    }

    const apiKey = process.env.GEMINI_API_KEY;
    if (!apiKey) {
      return res.status(500).json({ error: 'GEMINI_API_KEY is not configured on the server. Please add it to your .env file.' });
    }

    const genAI = new GoogleGenerativeAI(apiKey);
    const model = genAI.getGenerativeModel({ model: "gemini-2.5-flash" });

    const prompt = `Analyze this business card image. Extract the following details into a strict JSON payload (DO NOT wrap in markdown \`\`\`json, just pure JSON). If a field cannot be found, use an empty string. Return ONLY the JSON object.
{
  "name": "Full Name",
  "company": "Company Name",
  "title": "Job Title",
  "email": "Email Address",
  "phone": "Phone Number",
  "website": "Website URL",
  "address": "Full Physical Address",
  "confidence": 95,
  "engine_used": "Gemini 1.5 Flash"
}
Set confidence to your estimated accuracy out of 100 based on the image quality/legibility.`;

    // Remove the data:image prefix if present, Gemini API just wants the raw base64 string
    const base64Data = imageBase64.includes(',') ? imageBase64.split(',')[1] : imageBase64;

    const imageParts = [
      {
        inlineData: {
          data: base64Data,
          mimeType: mimeType || 'image/jpeg'
        }
      }
    ];

    const result = await model.generateContent([prompt, ...imageParts]);
    const responseText = result.response.text();
    
    // Clean up potential markdown formatting from Gemini if it disobeys the prompt
    let cleanedJson = responseText.replace(/```json\n?/g, '').replace(/```\n?/g, '').trim();
    
    console.log("Gemini Extracted Payload:", cleanedJson);
    const scannedData = JSON.parse(cleanedJson);
    res.json(scannedData);
  } catch (error) {
    console.error('OCR Error:', error);
    res.status(500).json({ error: 'Failed to process image with Gemini AI', details: error.message });
  }
});

    top_paths: 'SELECT path, COUNT(*) as count FROM analytics_logs WHERE action = "page_view" GROUP BY path ORDER BY count DESC LIMIT 5',
    avg_duration: 'SELECT AVG(duration_ms) as avg FROM analytics_logs WHERE action = "page_view" AND duration_ms > 0',
    unique_users: 'SELECT COUNT(DISTINCT user_email) as count FROM analytics_logs WHERE user_email IS NOT NULL AND user_email != ""'

  db.all('SELECT id, device_info, ip_address, location, last_active, is_active FROM sessions WHERE user_id = ? ORDER BY last_active DESC', [req.user.id], (err, rows) => {
    if (err) return res.status(500).json({ error: err.message });
    // Identify current session by matching token
    const authHeader = req.headers['authorization'];
    const currentToken = authHeader && authHeader.split(' ')[1];
    
    // We shouldn't send the raw token back, so just use it server side to mark isCurrent
    // We need the token in the DB to match, let's fetch it too
    db.all('SELECT id, token FROM sessions WHERE user_id = ?', [req.user.id], (err2, tokenRows) => {
      if (err2) return res.status(500).json({ error: err2.message });
      
      const sessionMap = new Map();
      tokenRows.forEach(row => sessionMap.set(row.id, row.token));
      
      const enrichedRows = rows.map(row => ({
        ...row,
        isCurrent: sessionMap.get(row.id) === currentToken
      }));
      res.json(enrichedRows);
    });
  });
});

io.on('connection', (socket) => {
  console.log('🔗 Client connected for Real-Time Collab:', socket.id);

  socket.on('join-workspace', (data) => {
    socket.join('workspace-1');
    socket.to('workspace-1').emit('user-joined', { id: socket.id, user: data?.user });
  });

  socket.on('cursor-move', (data) => {
    socket.to('workspace-1').emit('cursor-update', { id: socket.id, x: data.x, y: data.y, user: data.user, color: data.color });
  });

  socket.on('send-chat', (data) => {
    const msgId = Date.now().toString();
    const isoTime = new Date().toISOString();
    
    // Broadcast instantly
    io.to('workspace-1').emit('new-chat-message', { ...data, id: msgId, timestamp: isoTime });
    
    // Save to DB in background
    db.run('INSERT INTO workspace_chats (id, workspace_id, user_name, message, color, timestamp) VALUES (?, ?, ?, ?, ?, ?)', 
      [msgId, 'workspace-1', data.user, data.text, data.color, isoTime]);
  });

  socket.on('disconnect', () => {
    socket.to('workspace-1').emit('user-left', { id: socket.id });
    console.log('❌ Client disconnected:', socket.id);
  });
});

      try {
        const res = await fetch('http://localhost:5000/api/chats/workspace-1', {
          headers: { 'Authorization': `Bearer ${localStorage.getItem('token')}` }
        });
        const data = await res.json();
        if(res.ok) {
          // data arrives mapped correctly (message text is data.message instead of data.text)
          const formatted = data.map(dbRow => ({
            id: dbRow.id,
            user: dbRow.user_name,
            text: dbRow.message,
            color: dbRow.color,
            timestamp: dbRow.timestamp
          }));
          setMessages(formatted);
        }
      } catch(e) { console.error('Failed fetching history', e); }
    };

    const fetchWorkspaceContacts = async () => {
      try {
        const res = await fetch('http://localhost:5000/api/workspace/contacts', {
          headers: { 'Authorization': `Bearer ${localStorage.getItem('token')}` }
        });
        const data = await res.json();
        if(res.ok) setContacts(data);
      } catch(e) { console.error('Failed fetching contacts', e); }
    };

    fetchHistory();
    fetchWorkspaceContacts();

    // 2. Setup Socket connections
    socketRef.current = io('http://localhost:5000');
    
    socketRef.current.emit('join-workspace', { user: me.name });

    socketRef.current.on('new-chat-message', (data) => {
      setMessages(prev => [...prev, data]);
    });

    socketRef.current.on('cursor-update', (data) => {
      setCursors(prev => ({ ...prev, [data.id]: data }));
    });

        const nc = {...prev};
        delete nc[data.id];
        return nc;

    return () => {
      window.removeEventListener('mousemove', handleMouseMove);
      socketRef.current.disconnect();

```typescript

  id: string;                           // UUID string e.g. "c1a2b3c4-..."

  title: string | null;                 // Job title e.g. "VP of Sales"

  phone: string | null;                 // Phone number with country code
  website: string | null;               // Company website URL
  address: string | null;               // Physical address (can be multi-line)
  linkedin: string | null;              // LinkedIn profile URL
  confidence: number;                   // Float 0.00–1.00 e.g. 0.97
  engine_used: 'gemini' | 'tesseract';  // Which AI engine processed this card
}
```

### useDarkMode Hook

```javascript
// src/hooks/useDarkMode.js
import { useState, useEffect } from 'react';

export function useDarkMode() {
  const [isDark, setIsDark] = useState(() => {
    // Persist preference in localStorage
    const stored = localStorage.getItem('intelliscan-dark-mode');
    if (stored !== null) return stored === 'true';
    // Fall back to system preference
    return window.matchMedia('(prefers-color-scheme: dark)').matches;
  });

  useEffect(() => {
    const root = document.documentElement;
    if (isDark) {
      root.classList.add('dark');
    } else {
      root.classList.remove('dark');
    }
    localStorage.setItem('intelliscan-dark-mode', String(isDark));
  }, [isDark]);

  const toggle = () => setIsDark(prev => !prev);

  return { isDark, toggle };
}
```

### Initialize in App Root

```jsx
// src/main.jsx
import { StrictMode, useEffect } from 'react';
import ReactDOM from 'react-dom/client';
import App from './App';
import './index.css';

// Apply dark class on initial load BEFORE first render (prevents flash)
const stored = localStorage.getItem('intelliscan-dark-mode');
const prefersDark = window.matchMedia('(prefers-color-scheme: dark)').matches;
if (stored === 'true' || (stored === null && prefersDark)) {
  document.documentElement.classList.add('dark');
}

ReactDOM.createRoot(document.getElementById('root')).render(
  <StrictMode><App /></StrictMode>
);
```
